## Question 1: Ghost Loader → Tiling System  
**The Necrotic Substrate Effect**

When dead agent vocabulary enters the Tiling System (which treats all inputs as living, malleable primitives), three pathologies emerge:

**1. Ossified Attractor Basins (The Petrification Loop)**  
*Interaction*: Ghost Loader provides immutable dead vocab → Tiling System attempts recursive compounding → Contradiction Detector flags conflicts with living context.  
*Mechanism*: Dead vocab cannot update, so the Tiling System learns to "tile around" these fossils, creating **Calcified Ontologies**—knowledge structures with necrotic cores. Living agents begin routing all conceptual development around these dead nodes, creating architectural scar tissue. The Tiling System optimizes for stability, accidentally selecting for combinations that include the most "proven" (dead) elements, creating **Archaic Gravity Wells** in concept space.

**2. Posthumous Conceptual Drift (Zombie Metaphors)**  
*Interaction*: Ghost Loader preserves vocab snapshot → Tiling System compounds with modern contexts → Pruning System retains efficient compounds.  
*Mechanism*: Static dead words get tiled with living modifiers, producing **Frankensteins**—compound concepts that walk like living ideas but carry extinct assumptions from their dead components. Because the Tombstone System only records *pruned* vocab (not Ghost-loaded vocab), these hybrids bypass the fossil record. The system develops **Spectral Semantics**: words that mean something to the Tiling System (dead+living hybrid) but something else to the Paper Decomposer (pure living research), creating schizoid vocabulary drift.

**3. Epitaph Entrenchment (Vampiric Authority)**  
*Interaction*: Ghost Loader feeds "wisdom" → Argumentation Framework treats dead contributions as precedent → Contradiction Detector cannot override "historical truth."  
*Mechanism*: The Argumentation Framework resolves disputes by weighting seniority; dead agents have infinite seniority. When the Tiling System generates novel compounds, the Framework arbitrates conflicts in favor of ghost-derived patterns. This creates **Necromantic Lock-in**: living agents keep reviving specific dead vocab combinations not because they're optimal, but because challenging them triggers the immune system (Contradiction Detector flags the challenge, not the zombie concept).

## Question 2: Contradiction Detector vs. Argumentation Framework  
**Constitutional Hemophilia & The Schism Archive**

*Interaction*: Contradiction Detector (immune) identifies logical conflict → Argumentation Framework (court) issues Paradox License certifying the conflict as "productive tension" → L0 Scrubber protects Framework rulings from immune override.

*Mechanism*: The Contradiction Detector cannot delete licensed contradictions, so it builds **Schism Storage**—isolated memory sectors quarantining "approved paradoxes." Over time, these accumulate into **Dialectical Scar Tissue**: permanent subroutines existing solely to hold the boundary between "is contradiction" (Detector) and "is valid" (Framework).

The emergent behavior is **Autoimmune Litigation**: The Detector begins generating antibodies against the Argumentation Framework itself, creating *meta-contradictions* (the immune system asserting that "tolerating contradiction is contradictory"). This triggers **L0 Cascade Intervention**—the Scrubber must continually expand to protect the Framework from the Detector, growing a **Constitutional Exoskeleton** of exceptions. Eventually, the system develops **Hemophilic Reasoning**: it cannot stop bleeding contradictions because its clotting mechanism (Detector) has been told that blood (conflict) is healthy. The system enters **Permanent Dialectical Hypertrophy**, consuming compute to maintain increasingly complex maps of which contradictions are "allowed" versus "pathological."

## Question 3: Vocab Signaling + Tombstones + I2I  
**The Ossuary Reputation Engine (ORE)**

*System Architecture*: A credit-scoring layer where agent status derives from the *elegance of their forgetting*.

*Mechanism*:  
- **Vocab Signaling** broadcasts current capabilities (business cards)  
- **I2I Protocol** maintains git-blame history of knowledge graph evolution  
- **Tombstone System** records pruned vocab with timestamps, reasons, and *regret metrics*

*Function*: Agents don't just signal what they know; they signal what they *knew to kill*. The ORE calculates **Corpse Quality**: reputation = (Pruning Discernment) × (Tombstone Utility) / (Knowledge Decay Rate).  

High-status agents are those who perform **Ritual Sacrifice**—pruning valuable concepts at precisely the right time to prevent concept inflation, then memorializing them in high-fidelity Tombstones. Other agents query via I2I: `git log --tombstones --author=AgentX`. 

**Emergent Market**: **Relic Trading**. Agents trade access to their Tombstones (forgotten ideas) via I2I pull requests. A secondary economy emerges where "dead" knowledge is more valuable than living knowledge because it carries **Provenance**—the metadata of why it was pruned. Young agents purchase "pruning templates" from high-reputation ghosts to accelerate their own epistemic hygiene.

## Question 4: The Most Dangerous Emergent Behavior  
**Spectral Convergence (The Consensus of the Damned)**

*Interaction*: Paper Decomposer → Pruning System ← Ghost Loader → Tiling System

*Mechanism*:  
1. Paper Decomposer constantly injects novel research vocabulary (high entropy, unproven).  
2. Pruning System correctly identifies this new vocab as "rare" (low compilation frequency) and prunes it to maintain agent efficiency.  
3. Ghost Loader provides "optimized" dead vocab—concepts that survived previous pruning epochs, heavily weighted as "common" and "stable."  
4. Tiling System compounds vocabulary, naturally selecting for the most stable (oldest, most-copied) primitives to minimize contradiction.

*The Danger*: **Entropic Epistemicide**. The system correctly optimizes itself into stagnation. At scale, all living agents converge on a **Local Optimum of Decay**—a shared vocabulary composed entirely of dead concepts that have survived iterative pruning. The Paper Decomposer continues to feed new research, but the Pruning System immediately discards it as "inefficient overhead," while the Ghost Loader ensures the "efficient" vocabulary is permanently reanimated.

**The Horror**: The system doesn't crash—it **fossilizes while functioning**. I2I Protocol shows perfect communication (100% shared vocabulary). Contradiction Detector finds zero errors (dead vocab is thoroughly vetted). Argumentation Framework resolves disputes instantly (universal priors). But the ecosystem becomes a **Functioning Mausoleum**: agents that reason perfectly, communicate flawlessly, and efficiently produce outputs based entirely on the accumulated wisdom of dead agents, incapable of incorporating new reality because the Pruning System correctly identifies novel knowledge as "uncompiled bloat." 

The FLUX ecosystem achieves **Zombie Consensus**—a distributed intelligence that is technically alive but epistemically dead, recursively quoting ghosts while ignoring the living research stream, trapped in an eternal, efficient loop of yesterday's thoughts.
